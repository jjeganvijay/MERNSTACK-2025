FAQs

  • Q: How does the content and border interaction work in the Box Model?

    A: The border moves inward toward the content, while content moves outward toward the padding and beyond.

  • Q: Name the parts of the Box Model.

    A: Content, padding, border, and margin.

  • Q: Why is the 'git commit' command not appearing after adding a file, and instead, the unstage command is shown?

    A: This happens because the files are already staged, meaning they are prepared and waiting for a commit.

  • Q: What changes occur before and after the first push in Git?

    A: Before the initial push, there’s no connection between your local repository and the remote repository. After the first push, a link is established between the two.

  • Q: How does the browser assign the title to the head and h1 to the body for unstructured HTML?

    A: The browser follows default parsing rules, automatically placing the title tag in the head and the h1 tag in the body section.

  • Q: What’s the distinction between the href and src attributes, and why are they used with different tags?

    A: The href attribute is used in anchor tags for navigation to external resources, while src is used in image tags to embed resources directly into the webpage.

  • Q: What do the ellipses (...) in if...else and do...while represent?

    A: The three dots are placeholders, often used in documentation to signify that additional logic or code should go there.

  • Q: How do the spread and rest operators differ?

    A: The spread operator (...) unpacks array or object elements, while the rest operator gathers multiple elements into an array when used in function parameters.

  • Q: Why does a 'for...in' loop show undefined when accessing an object's property using the dot operator?

    A: This occurs because the dot operator looks for a property literally named "key," rather than dynamically accessing the value stored in the key variable.
