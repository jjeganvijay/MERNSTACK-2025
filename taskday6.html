Why Components Start with Capital Letters

In JavaScript frameworks like React, it's a best practice to use uppercase letters for naming components. This practice ensures that components are easily distinguishable from standard HTML tags.

- Component Naming Rules: When working with JSX (JavaScript XML), it's required that custom components use an uppercase first letter, such as CustomComponent. This allows React to recognize them as components, unlike native HTML tags like <div>, which are lowercase.

- The Reason for Capitalization: If a component is named using lowercase letters (e.g., <customcomponent>), React assumes it is a standard HTML tag, which can lead to unintended behavior when rendering.

The Purpose of Default Export

The default keyword in ES6 modules enables you to export a single main value or entity from a file, simplifying module usage.

- What Does Default Export Mean? Using export default signifies that the module has one primary export. For example, in React, you typically use default export to export a component as the primary entity. 

export default CustomComponent;

- Advantages of Default Export: Default exports allow you to import the exported entity without needing to use its exact name, offering a cleaner and more flexible import syntax. 

import CustomComponent from './CustomComponent';

This approach makes the import process easier, as you don't have to remember the original exported name precisely.
